渐进增强与优雅降级
1、渐进增强
  这个是指在开发过程中，先构建针对低版本浏览器的基本功能，然后再此基础上，再向高版本的浏览器或设备扩展一些效果、交互或其他追加功能。
2、优雅降级
  一开始就构建完整功能的站点，然后针对浏览器进行测试和修复（hack）,使其可以向下兼容低版本浏览器
  在开发过程中，选择渐进增强还是优雅降级，取决于企业的用户客户端的版本。如果低版本用户居多，当然优先采用渐进增强的开发流程；
  如果高版本用户居多，为了提高大多数用户的使用体验，那当然优先采用优雅降级的开发流程。
.transition { /*渐进增强写法*/
  -webkit-transition: all .5s;
     -moz-transition: all .5s;
       -o-transition: all .5s;
          transition: all .5s;
}
.transition { /*优雅降级写法*/
          transition: all .5s;
       -o-transition: all .5s;
     -moz-transition: all .5s;
  -webkit-transition: all .5s;
}
